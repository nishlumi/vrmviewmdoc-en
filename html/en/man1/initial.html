1. Introduction

This application was developed with the aim of making it easy to create and play animations with VRoid (VRM) like MMD. With this app, you can easily do the following things.

  • You can take a screenshot by letting VRM take your favorite pose

  • Animation can be created by moving VRM from scratch without using MMD, Unity editor, Blender, etc.

We did not aim for a full-fledged 3D animation application, so it is not suitable for professional use. And animations made with this application can only be played with this application (for now). [1] However, it should be much easier than creating directly on the Unity editor. Also, since it is a web application, it can be used anywhere with a web browser. (Some features have technical limitations.)

[1]

From ver 2.0, we started to support exporting in .anim format on a trial basis.

1.1. What is VRoid (VRM)?

VRoid (VRM) is a new, easy-to-use 3D model standardized for use in VR applications and 3D games. VRM is widely open to the public with better specifications than the famous MMD, which contributed greatly to the world as a 3D model standard and animation tool, and is easy to use for general purposes. And the development and operation are actively continued, and the support side is also perfect.

It seems that the number of compatible apps is comparable to the MMD model, and now it is developing a variety of fields such as smartphone apps such as Android / iOS, PC and VR device applications.

It seems that it was originally created assuming the avatar of the VR application, but it is also possible to 3D animation with VRM alone. With the game creation environment called Unity, even at an individual level, you can use Unity’s animation function to move VRM limbs and pose and animate him.

However, it is quite difficult to introduce Unity, introduce a VRM loading library, and handle Unity’s animation function. If you don’t use assets such as VeryAnimation that make it easier to create animation (even if you do), the threshold will be high.

Based on the current situation, I personally think that the biggest disadvantage of VRM is that there are still few software that can match MMD as software. (Even if there is, it is troublesome to have to create motion with exaggerated tools such as Unity and other animation tools…)

So, for this application, I tried various animation creation support assets such as VeryAnimation in Unity and gave up, so I decided to contribute to the world by creating an easy-to-use VRM operation library and animation creation application. was the start.
